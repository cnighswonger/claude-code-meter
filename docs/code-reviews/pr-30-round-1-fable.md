## What Needs Attention

1. **Spec contradiction: the proposed schema cannot pass the proposed test plan.** The Scope section specifies exactly two plain `.optional()` fields (~10 LOC, "drop into the existing strict-object pattern"), but the test plan requires "row with `agent_id_source` set but `agent_id` absent **fails** validation." Two independent optionals cannot produce that failure â€” Zod will happily validate a source-without-value row. Enforcing the asymmetric invariant (source â‡’ value, but value alone OK) requires a `.superRefine()`/`.check()` on the object. The two readings produce *different wire contracts*, and this is a load-bearing wire-format PR, so the directive text must pick one before implementation:
   - **Option A (recommended): add the refine.** Amend Scope to name it explicitly. Note: chaining a refinement onto `z.strictObject` wraps the type (ZodEffects); I verified nothing in `src/` consumes `MeterRowSchema.shape`/`.extend`, so this is safe today, but the directive should record that constraint.
   - **Option B: drop the failing test** and document "both-or-neither is producer-side discipline; the schema does not enforce it" with rationale. Defensible â€” the only known producer (cache-fix `usage-log.mjs`) sets both fields in the same branch, so a source-only row is an emission bug that can't occur from the specified producer.
   Either is acceptable; an unresolved contradiction is not. This is the one item I'd want amended in the directive before AITL implements.
2. **Enum value casing breaks the meter's own convention â€” flag before it ossifies.** Every existing enum/enum-like value in this schema file is snake_case or bare lowercase: `five_hour`/`seven_day` (`qrepresentative_claim`, `representative_claim_distribution`), `max_5`/`max_20` (`plan_tier`), `standard`/`fast` (`speed`), and `qstatus`/`qclaim` regexes are `[a-z_]*` â€” hyphens aren't even representable there. `"cc-header"`/`"cache-fix-derived"` introduces kebab-case into a snake_case value space. 3. **Failure-mode loudness of the attestation breach is marginal.** The live ingestion path is the tailer, and its rejection of unknown-key rows is a debug-level log (visible only under `CLAUDE_METER_DEBUG=1`) plus a `skipped=N` counter in `claude-meter ingest` tick output (`src/cli/ingest.mjs`). 4. **Test plan should mirror the `request_id` precedent suite fully.** The PR #24 suite (`test/schema-request-id.test.mjs`) includes cases the directive's list omits: the 64-char boundary accept + 65-char reject pair, non-string type rejection (`123`, `null`, `true`, array, object), and the unknown-sibling-key strictness-preserved test.
